# 🤖 AI-Кротик: Telegram Bot с поддержкой ИИ и Поиском

🏠 Корневые файлы
- `bot.js` — Ядро бота. Инициализация Telegraf, точка входа для всех сообщений, координация всех модулей.
📁 modules/ — Основная логика
**🔧 Системные модули**
- `config.js` — Конфигурация. Токены API (.env), ID админа, пути к файлам БД, реестр моделей ИИ (OpenRouter/HuggingFace).
- `storage.js` — База данных. Работа с JSON-файлами, автоматическая очистка истории (TTL), подсчет пользователей.
- `menus.js` — Интерфейс пользователя. Все меню, динамические кнопки с индикаторами, админ-панель с количеством подписчиков.
- `utils.js` — Вспомогательные функции. Извлечение URL, трассировка ID файлов, отправка PDF, обработка платежей.
**🎮 Обработчики**
- `handlers.js` — Контроллер навигации. Обработка нажатий кнопок меню, маршрутизация между разделами.
- `commands.js` — Системные команды. Обработка /start, /broadcast, /search, /logs, /help.
- `actions.js` — Callback-обработчики. Все bot.action для заказов, целей Ба Цзы, воронки продаж.
- `mapPath.js` - утилита для перехода к описанию услуги карта Пути
- `keyboards.js` - динамическое создание кнопок в воронке funnel.js
**🤖 Искусственный интеллект**
- `ai.js` — Диспетчер ИИ. Переключение между OpenRouter, Hugging Face и поисковыми запросами.
- `ai-helper.js` — Помощник ИИ. Функция askAI с автоматическим перебором моделей при ошибках.
- `search.js` — Веб-поиск. Интеграция с Serper API (Google) для получения актуальных данных.
- `roles.js` — Роли для моделей. Предопределенные роли и настройки для различных типов запросов.
**📧 Рассылка и уведомления**
- `broadcast.js` — Сервис массовой рассылки. Отправка фото, документов и текста всем пользователям.
- `broadcastHandler.js` — Пошаговый сбор контента. Стейт-машина для создания рассылок с предпросмотром.
- `reminders.js` - планировщик задач
**💳 Платежи и оплата**
- `payments.js` — Управление ручной оплатой. Бронирование цен, обработка чеков, отслеживание состояния пользователей.
- `receipts.js` — Обработка входящих чеков от пользователей (фото и документы).
**🎯 Маркетинговые воронки**
- `funnel.js` — Интерактивная воронка «Личная Сказка». Автоматизация продаж через геймификацию.
- `astro.js` — АстроЧекап (ЛИД-МАГНИТ). Интерпретация карты личности по системе Ба Цзы через ИИ.
📁 data/ — Хранилище данных
- `settings.json` — Профили пользователей. Выбранная модель ИИ, режим ответа, состояние воронок.
- `history.json` — История диалогов. Контекст общения с ограничением по времени и токенам.
- `broadcasts.json` — История рассылок. Статистика выполненных кампаний.
- `funnel_config.json` — Конфигурация воронки. Настройки для интерактивной сказки.
- `questions.json` — Вопросы пользователей. Сбор обратной связи и запросов.


### 🛠 Логика обработки сообщения
1. Инициализация пользователя
Проверка наличия пользователя в БД
Создание профиля при первом обращении
Загрузка истории и настроек
2. Фильтрация админских функций
Проверка режима рассылки
Трассировка ID медиафайлов (только вне режима рассылки)
Обработка команд администратора
3. Маршрутизация
handlers.js анализирует текст сообщения
Определение типа запроса (меню, команда, контент)
Перенаправление в соответствующий обработчик
4. Обработка ИИ
hf: префикс → Hugging Face API
search: префикс → Поиск + ИИ анализ
По умолчанию → OpenRouter API
5. Маркетинговые воронки
АстроЧекап для лидогенерации
Интерактивная сказка для конверсии
Автоматическое перенаправление на специалиста

### 📦 Основные зависимости
- `telegraf` — Работа с Telegram Bot API.
- `axios` — HTTP-запросы к нейросетям и поисковым сервисам.
- `@huggingface/inference` — Официальный SDK для работы с моделями Hugging Face.
- `dotenv` — Безопасное хранение ключей в файле `.env`
- `node-schedule` - библиотека (планировщик задач) для Node.js, которая позволяет запускать код в строго определенное время или через заданный интервал.

### 🌪 Интерактивная воронка «Личная Сказка» (`modules/funnel.js`)
 модуль для автоматизации продаж через геймификацию и психологические практики.

**Особенности:**
- **Символдрама в PDF**: Автоматическая выдача подарка (мини-практики) в формате документа.
- **МАК-карты (MediaGroup)**: Демонстрация 9 метафорических образов одним альбомом.
- **Интерактивный выбор**: Обработка callback-запросов для каждой карты с персональной интерпретацией.
- **Конверсионные кнопки**: На каждом этапе встроена прямая ссылка на специалиста (`@sherab_wangmo`).

**Особенности:**
- калькулятор Ба Цзы
- передача расчета нейросети с настроенной ролью ответа
- переход к платным консультациям в конце воронки

**🎯 Команды администратора**
/start — Главное меню с количеством подписчиков
/broadcast — Создание и отправка рассылки
/search — Поиск информации через ИИ
/logs — Просмотр истории рассылок
/help — Справочная информация
**📈 Мониторинг и аналитика**
Автоматический подсчет активных пользователей
Логирование всех рассылок с метриками
Отслеживание ошибок доставки
Статистика использования воронок
**Логика оплаты**
-  Пользователь нажимает кнопку оплаты ("Забронировать" или "Оплатить онлайн")
- В payments.js отправляется сообщение с реквизитами
Показывается кнопка "✅ Я оплатил(а)"
Пользователь нажимает "✅ Я оплатил(а)"
- В actions.js устанавливается флаг: awaitingReceipt: true
Бот просит прислать чек: "Пришлите скриншот чека..."
- Пользователь отправляет фото/документ (чек)
- Если awaitingReceipt: true → receipts.js обрабатывает чек
- Чек отправляется админу, пользователь получает подтверждение
- Флаг сбрасывается: awaitingReceipt: false
Пользователь уходит в другие разделы меню - В handlers.js флаг awaitingReceipt сбрасывается, Ожидание чека прекращается

Админ отправляет фото/документ (для трассировки ID)
Если НЕ ожидается чек → utils.js показывает ID файла
Если ожидается чек → обрабатывается как чек пользователя
**Состояния пользователя:**
{
  "userId": {
    "awaitingReceipt": true/false,     // Ожидание чека
    "paymentStep": "waiting_receipt" | "receipt_received", // Этап оплаты
    "reserved": true,                  // Бронирование депозита
    "fullPaymentStarted": true         // Начало полной оплаты
  }
}
