# 🤖 AI-Кротик: Telegram Bot с поддержкой ИИ и Поиском

### 📂 Структура проекта
- `bot.js` — **Ядро**. Инициализация Telegraf, входная точка для всех сообщений и координация модулей.
- `modules/` — **Логика**:
  - `config.js` — Конфигурация: токены (.env), ID админа, пути к БД и реестр моделей (OpenRouter/HF).
  - `storage.js` — БД бота. Чтение/запись JSON и **автоматическая очистка истории** (TTL).
  - `menus.js` — **UI/UX**. Описание всех меню, динамические кнопки с ✅ и админ-панель.
  - `handlers.js` — **Контроллер**. Навигация по разделам «Услуги», «Тренинги» и выбор режимов ИИ.
  - `commands.js` — Системные команды: `/start`, `/broadcast` (рассылка), `/logs`.
  - `ai.js` — **Диспетчер ИИ**. Переключает запросы между OpenRouter, Hugging Face и Поиском.
  - `search.js` — **Веб-поиск**. Интеграция с Serper API (Google) для получения свежих данных.
  - `broadcast.js` — Сервис массовой рассылки с поддержкой фото и текста.
- `data/` — **Хранилище**:
  - `settings.json` — Профили пользователей (выбранная модель, режим ответа).
  - `history.json` — Контекст диалогов (ограничен по времени и количеству токенов).
  - `broadcasts.json` — История и статистика выполненных рассылок.

### 🛠 Логика обработки сообщения
1. **Админ-фильтр**: Проверка, не находится ли админ в режиме создания рассылки.
2. **Маршрутизатор**: `handlers.js` проверяет, является ли текст командой меню (Услуги, Режимы и т.д.).
3. **Диспетчер ИИ**:
   - Если модель с префиксом `hf:`, запрос уходит на Hugging Face.
   - Если с префиксом `search:`, сначала вызывается `search.js`, затем результат передается в ИИ.
   - В остальных случаях используется OpenRouter.

### 📦 Основные зависимости
- `telegraf` — Работа с Telegram Bot API.
- `axios` — HTTP-запросы к нейросетям и поисковым сервисам.
- `@huggingface/inference` — Официальный SDK для работы с моделями Hugging Face.
- `dotenv` — Безопасное хранение ключей в файле `.env`
